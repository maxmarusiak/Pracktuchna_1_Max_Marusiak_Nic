Інкапсуляція — приховування внутрішнього стану об’єкта та доступ до нього через властивості/методи. Приклади: приватні поля _fullName, _dateOfBirth, _recordBookNumber; валідація у FullName; перевірка діапазону в AverageGrade; read-only Age; приватний список _students у StudentGroup.

Авто-властивості використовував для простих даних без логіки (Notes, Status, EnrollmentDate). Повні властивості — коли потрібна валідація або обчислення (FullName, RecordBookNumber, AverageGrade, Email, Age).

Валідація застосована для FullName, RecordBookNumber, AverageGrade, Email, DateOfBirth. Використані види: перевірка довжини, формат (Regex), діапазон, коректність дати, винятки.

Read-only властивості потрібні для значень, які не змінюються після створення об’єкта (Age). Init-only — для значень, які встановлюються лише під час ініціалізації (FullName, RecordBookNumber).

Age обчислювана, бо: вік змінюється щодня; збережене поле швидко стане неактуальним; обчислення гарантує актуальний результат. Код: public int Age => CalculateAge();

Принцип єдиної відповідальності: Student — дані + валідація; StudentGroup — управління списком студентів; GradeJournal — оцінки та середній бал; Program — точка входу.

Труднощі: розподіл логіки між класами, вибір типів властивостей, організація валідації. Рішення: SRP, обчислювані властивості, валідація в setter/init, чітка структура класів.
